### 二、 详细实现方案

#### 1. 用户注册与后台分析流程

**步骤 1：客户端上传**
*   用户通过 App 或 Web 拍摄/选择本人清晰照片。
*   **前端预检**：在前端进行简单的人脸框检测，确保画面中有且仅有一张人脸，且光照充足，避免无效流量冲击服务器。

**步骤 2：图像质量与合规性检查**
*   后端接收到图片后，调用 AI 服务接口。
*   **质量检测**：检测模糊度、光照、角度。
*   **活体检测（静默）**：防止用户上传他人照片或打印照片注册。
*   若不合格，返回错误信息（如：人脸模糊，请重新上传）。

**步骤 3：特征提取**
*   通过深度学习模型（如 ArcFace, CosFace, RetinaFace）提取人脸的 512 维或 1024 维特征向量。
*   **注意**：后端不直接存储原始高清人脸照片用于比对，而是存储这个“特征向量”。

**步骤 4：数据落库**
*   **关系数据库**：存储用户 ID、姓名、部门、权限等级、注册时间。
*   **向量数据库**：存储 `User_ID` <-> `Feature_Vector` 的映射关系。
*   **对象存储 (OSS)**：归档存储原始人脸照片（用于审计或后续重训）。

#### 2. 员工刷脸开门流程

该流程对**实时性**要求极高，需毫秒级响应。

**步骤 1：视频流抓拍**
*   门禁终端通过 RTSP 或 WebSocket 实时将视频帧推送到算力节点。
*   （或者采用边缘计算方案：算法部署在门禁终端本地，仅需上传结果）。

**步骤 2：人脸检测与活体检测**
*   **人脸检测**：定位画面中的人脸位置。
*   **活体检测（动态）**：使用红外双目摄像头或 3D 结构光，配合算法判断是否为真人（防电子屏幕攻击、防3D面具）。这是安全性的核心。

**步骤 3：特征比对 (1:N Search)**
*   提取当前帧的人脸特征向量。
*   在向量数据库中进行 **1:N 检索**（N为注册人数）。
*   计算余弦相似度。例如设定阈值 `Score > 0.85` 则判定为同一人。

**步骤 4：权限判定与开门**
*   若匹配成功，查询该用户是否有当前时间段进入权限。
*   通过 **MQTT 协议** 或 **WebSocket** 向门禁终端发送“开门”指令。
*   终端收到指令 -> 驱动 GPIO 高电平 -> 继电器闭合 -> 电磁锁通电开门。
*   屏幕显示“欢迎，张三”及语音播报。

**步骤 5：日志记录**
*   异步记录开门记录（时间、地点、抓拍图、相似度得分）到数据库，用于后续考勤报表生成。

### 三、 关键技术选型与硬件建议

#### 1. AI 算法技术栈
*   **开发语言**：Python (模型推理) + Go/Java (业务逻辑)。
*   **核心框架**：
    *   人脸检测：**RetinaFace** 或 **YOLO-Face** (速度快，精度高)。
    *   人脸识别：**ArcFace** (目前识别率业界领先)。
    *   活体检测：**Silent-Face-Anti-spoofing** (静默活体) 或配合红外/深度相机硬件。
*   **向量检索引擎**：
    *   **Milvus** 或 **Faiss**：专门用于海量高维向量的相似度检索，比 MySQL 模糊查询快数千倍，适合万人级以上规模。

#### 2. 硬件选型建议
*   **小规模场景（<100人）**：
    *   树莓派 4B / Jetson Nano + USB 摄像头 + 继电器板。
    *   方案：本地跑轻量级模型（如 MobileFaceNet），速度约 200ms/帧。
*   **中大规模场景（商业写字楼）**：
    *   工业级安卓/Linux 人脸一体机（含咪头来客显示）。
    *   算力：建议 NPU 算力 > 1 TOPS，支持本地实时活体检测。
    *   门锁控制：韦根接口或继电器输出连接门禁电源。

#### 3. 安全性设计（非常重要）
*   **传输加密**：视频流与控制指令必须走 TLS 加密通道，防止流量劫持。
*   **数据脱敏**：原始人脸照片需加密存储在 OSS，查看日志时需打码处理以符合隐私法规。
*   **防重放攻击**：开门指令需带时间戳和签名，终端校验时间戳有效期（如 5秒内有效），防止截获数据包重放开锁。
